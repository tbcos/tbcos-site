# WEBSITE_BUILD_GUIDE

## 문서 목적

이 문서는 `THREE BODY / TBcos` 웹사이트를 실제로 구축하면서 정리된 구조, 설정, 운영 방식, 그리고 AI와 더 효율적으로 협업하는 방법을 함께 담은 실무 가이드입니다.

이 문서를 보면 아래 내용을 빠르게 파악할 수 있습니다.

- 현재 웹사이트가 어떤 구조로 만들어져 있는지
- 배포와 도메인 연결이 어떻게 되어 있는지
- SEO와 검색엔진 등록은 어디까지 되었는지
- 문의 폼과 로그인 기능은 어떤 방식으로 연결되어 있는지
- 앞으로 수정 요청을 어떻게 하면 더 빠르고 효율적으로 진행할 수 있는지

## 1. 프로젝트 개요

### 사이트 목적

`TBcos` 브랜드와 `THREE BODY` 회사 정보를 소개하고, 제품 라인업, 제조 역량, 문의 접수, 향후 파트너 로그인 기능까지 확장 가능한 기업 웹사이트를 운영하는 것이 목적입니다.

### 현재 운영 주소

- 메인 주소: `https://www.tb-cos.kr`

### 기술 구성

- 정적 웹사이트: `HTML / CSS / JavaScript`
- 스타일 구조: `기존 CSS + Tailwind 중심으로 점진 전환`
- 배포: `GitHub Pages`
- 도메인 관리: `GoDaddy`
- 문의 폼 저장: `Google Apps Script + Google Sheets`
- 로그인 구조: `Supabase Auth`

## 4. 다국어 운영 방식

현재 사이트는 `영어 기본`, `한국어 전환` 구조입니다.

### 적용 범위

- 메인 페이지
- 로그인 / 회원가입 페이지
- 계정 페이지
- 인증 콜백 페이지

### 운영 원칙

- 영어와 한국어는 같은 내용을 다른 언어로 전달해야 함
- 한쪽만 수정하면 다른 언어도 함께 맞춰야 함
- 법적 기준 문서는 한국어가 우선이며 영문은 참고본

### 법적 문서 우선순위

- 기준 문서: `privacy.html`, `terms.html`
- 참고용 문서: `privacy-en.html`, `terms-en.html`
- 문서 충돌 시 한국어 버전 우선

## 14. AI와 효율적으로 협업하는 방법

이 부분은 앞으로 토큰 사용량을 줄이고, 더 빠르게 수정하기 위한 실제 운영 가이드입니다.

### 가장 중요한 원칙

전문 용어를 정확히 몰라도 괜찮습니다.  
짧고 자연스럽게 요청해도 충분합니다.

가장 효율적인 요청 방식은 아래 3가지만 말하는 것입니다.

- 어디를 바꾸고 싶은지
- 어떻게 바꾸고 싶은지
- 어떤 느낌을 원하는지

예시:

- `네비게이션 로고를 더 크게 해줘.`
- `모바일에서 제품 카드 여백 없애줘.`
- `이 문구를 더 고급스럽게 바꿔줘.`
- `영어/한국어 둘 다 맞춰줘.`

### 특히 효율이 좋은 요청 문장

- `바로 적용해줘`
- `설명 짧게`
- `옵션 3개만 줘`
- `실행하지 말고 설명만`
- `비교만 해줘`
- `모바일 기준으로 잡아줘`
- `영어/한국어 같이 수정해줘`

### 앞으로 추천하는 협업 방식

1. 사용자는 짧게 요청
2. AI는 필요한 경우에만 짧게 확인
3. 대부분은 바로 실행
4. 긴 설명은 요청받았을 때만 제공
5. 선택지가 필요하면 2~3개만 제안

즉, 사용자는 자세히 설명하려고 애쓰지 않아도 됩니다.  
짧게 말하고, AI가 문맥을 해석해 실행하는 방식이 가장 효율적입니다.

## 15. 자주 사용할 수 있는 요청 예시

- `이 문장 어색해. 더 자연스럽게`

- `회사 소개 톤에 맞게 바꿔줘`
- `영어/한국어 같은 의미로 맞춰줘`

### 운영 / 배포

- `배포 기준으로 다시 점검해줘`
- `SEO 상태 체크해줘`
- `검색엔진 등록 상태 확인해줘`
- `실사이트 기준으로 검증해줘`

### 설명 방식 제어

- `짧게 설명해줘`
- `초보자 기준으로 설명해줘`
- `설명보다 바로 실행해줘`
- `실행하지 말고 우선 알려줘`

## 16. 이 문서를 어떻게 활용하면 좋은가

이 문서는 아래 목적에 맞게 계속 업데이트하면서 쓰면 좋습니다.

- 웹사이트 운영 기록 문서
- 차후 수정 요청 기준 문서
- 외부 협업자 전달용 프로젝트 개요 문서
- AI 협업 방식 가이드

추천 방식:

- 큰 기능이 추가되면 문서 업데이트
- 배포 구조가 바뀌면 도메인/배포 섹션 수정
- 로그인/문의폼 구조가 변경되면 관련 섹션 갱신
- 자주 쓰는 요청 패턴이 생기면 예시 섹션에 추가
