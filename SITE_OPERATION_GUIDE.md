# 사이트 실무 운영 가이드

이 문서는 TBcos 웹사이트를 실제로 운영할 때 필요한 실무 기준을 정리한 문서입니다.

---

## 1. 운영 목적

현재 사이트의 운영 목적은 아래 4가지입니다.

1. 회사 소개
2. 제품 및 브랜드 소개
3. 문의 접수 및 대응
4. 회원/파트너 계정 관리

즉, 운영의 핵심은 아래 두 가지입니다.

- 문의를 놓치지 않는 것
- 회원과 사이트 상태를 안정적으로 유지하는 것

---

## 2. 핵심 운영 항목

### A. 문의 관리

현재 문의 폼은 Supabase `public.inquiries` 테이블에 직접 저장됩니다.

운영자가 확인할 항목:

- 새 문의가 들어왔는지
- 문의 내용이 정상 저장됐는지
- 처리해야 할 문의가 남아 있는지
- 테스트 문의와 실제 문의가 구분되는지

권장 상태값 예시:

- `new` : 새 문의
- `in_progress` : 검토 중
- `replied` : 답변 완료
- `closed` : 종료

현재는 상태 관리 전용 UI가 없으므로, 우선은 문의가 정상 저장되는지와 답변 여부를 중심으로 운영합니다.

---

### B. 회원 관리

회원 기능은 Supabase Auth 기반으로 운영됩니다.

운영자가 확인할 항목:

- 신규 회원 가입 여부
- 승인 대기 회원 여부
- 테스트 계정 여부
- 로그인 또는 비밀번호 재설정 문제 발생 여부

현재 기본 구조:

- 신규 회원 상태: `pending`
- 운영 판단 후:
  - `approved`
  - `rejected`

---

### C. 사이트 정상 동작 확인

운영자는 실제 사용자 입장에서 사이트를 주기적으로 확인해야 합니다.

점검 항목:

- 메인 페이지 접속 가능 여부
- 모바일 메뉴 정상 동작 여부
- 로그인 페이지 접속 가능 여부
- 회원가입 기능 동작 여부
- 비밀번호 찾기 기능 동작 여부
- 카카오 로그인 진입 여부
- 문의 폼 전송 가능 여부

---

### D. 검색엔진 상태 확인

검색 노출 상태는 한 번 등록하고 끝나는 것이 아닙니다.

확인할 항목:

- Google Search Console 오류 여부
- 네이버 서치어드바이저 오류 여부
- `robots.txt` 정상 여부
- `sitemap.xml` 정상 여부
- 색인 상태 변화 여부

---

## 3. 운영 주기 추천

### 매일 또는 문의가 자주 들어올 때

- Supabase `inquiries` 테이블에서 새 문의 확인
- 필요한 문의 답변 진행

### 주 1회

- 사이트 실제 접속 테스트
- 로그인/회원가입/비밀번호 재설정 점검
- 모바일 화면 점검
- 회원 목록 및 상태 확인

### 월 1회

- 검색엔진 상태 확인
- 문구/제품 정보 최신성 점검
- 테스트 데이터 정리
- Git 상태 및 백업 지점 확인

---

## 4. 문의 운영 절차

권장 기본 흐름:

1. 문의 접수
2. 문의 확인
3. 내부 검토
4. 답변
5. 완료 처리

운영 팁:

- 테스트 문의는 메시지 앞에 `[TEST]`를 붙여 구분합니다
- 실제 운영 문의와 테스트 문의를 혼동하지 않도록 정리합니다

---

## 5. 회원 운영 절차

권장 기본 흐름:

1. 회원 가입 확인
2. 이메일 및 회사명 확인
3. 파트너/바이어 성격 확인
4. 승인 또는 반려 판단

현재는 Supabase에서 직접 관리하는 형태입니다.

관리 위치:

- `Supabase > Authentication > Users`

---

## 6. 운영자가 자주 확인하는 위치

실무 운영 시 자주 열게 되는 곳은 아래와 같습니다.

1. Supabase `Table Editor > inquiries`
2. Supabase `Authentication > Users`
3. Google Search Console
4. 네이버 서치어드바이저
5. GitHub 저장소

---

## 7. 수정 작업 시 원칙

사이트 수정 시 아래 원칙을 지키는 것이 좋습니다.

### 문구 수정

- 한국어와 영어를 함께 점검
- 한쪽만 바꾸지 않기

### 기능 수정

- 수정 전 Git 커밋 또는 현재 상태 백업
- 수정 후 실제 배포 주소에서 확인

### 테스트 데이터

- 테스트 메시지에는 `[TEST]` 표기
- 운영 완료 후 테스트 데이터 정리

### 모바일 점검

- iPhone, Android 모두 가끔 확인
- 햄버거 메뉴, 로그인, 문의 폼은 특히 모바일 기준으로 점검

---

## 8. 현재 운영 구조 요약

### 문의

- 저장 위치: Supabase `public.inquiries`
- 공개 문의폼에서 직접 저장

### 회원

- 인증: Supabase Auth
- 이메일 메일 발송: iCloud SMTP
- 카카오 로그인: 설정 완료

### 배포

- GitHub Pages
- 운영 주소: `https://www.tb-cos.kr`

---

## 9. 현재 남은 운영 개선 과제

현재 사이트 기준으로 다음 과제가 남아 있습니다.

1. 모바일 실사용 최종 점검
2. 검색엔진 색인 반영 여부 확인
3. 관리자용 회원 관리 페이지 기획 및 구현
4. 문의 관리 UI 추가 여부 검토
5. SNS 실제 링크 연결
6. 공유용 대표 썸네일 이미지 제작

---

## 10. 향후 추천 순서

운영 효율을 높이기 위한 다음 추천 순서:

1. 관리자용 회원 관리 페이지
2. 문의 목록/상태 관리 페이지
3. 검색엔진 상태 최종 확인
4. SNS/공유 이미지 정리

---

## 11. 한 줄 운영 기준

이 사이트의 실무 운영 핵심은 아래와 같습니다.

- 문의를 놓치지 않는다
- 회원 상태를 관리한다
- 사이트가 항상 정상 작동하는지 확인한다
- 수정 전후에는 Git으로 안전하게 관리한다
